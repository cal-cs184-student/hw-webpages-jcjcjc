Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm works by repeatedly lerping (linearly interpolating) between pairs of neighboring control points until you are left with one point. Given a parameter t, you take your n control points, lerp each neighboring pair to get n-1 new points, then repeat on those new points, and keep going until you have just one point left. That point falls on the Bezier curve at parameter t. Changing t from 0 to 1 traces out the full curve.

For the implementation, evaluateStep does one round of this algorithm. It loops through the input points, lerps each neighboring pair using (1-t)*p[i] + t*p[i+1], and returns the resulting list of intermediate points. To evaluate the full curve you would simply call this method repeatedly until one point remains.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends from curves to surfaces by applying the 1D algorithm across both directions. Given a grid of control points, each row of control points define a curve. We can recursively use 1D de Casteljau's algorithm to find a single point on that lies on each curve at parameter u. Doing this for each row gives us a set of control points on a "moving curve" that lie roughly along a column. We can again use 1D de Casteljau's algorithm with parameter v on this set of points to find a final surface point that lies on the Bezier at (u, v).

In terms of code, we have a method evaluateStep that computes a single step of de Casteljau's algorithm for a set of 3D points and parameter t by lerping adjacent points together. The method evaluate1D recursively calls evaluateStep on a set of 3D points and outputs a single point that lies on the "moving curve". The evaluate method outputs the Bezier surface point by calling evaluate1D on each row of points with parameter u, then calling evaluate1D on our resulting column of control points with parameter v.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

For area-weighted vertex normals, we iterate over all triangles incident to the vertex using the half-edges. Starting from the vertex's outgoing half-edge, we traverse neighboring faces in a loop using h = h->twin()->next() until we return to the original half-edge. In the loop, we identify three vertices: the current vertex (vertex1), the vertex at the other end of the current half-edge (vertex2 = h->twin()->vertex()), and the third vertex of that triangle (vertex3 via h->twin()->next()->next()->vertex()).

We then compute the cross product of the two edge vectors (vertex2 - vertex1) and (vertex3 - vertex1). The cross product gives an area-weighted normal because its magnitude is equal to twice the triangle's area. We ran into the issue where our normals were pointing to the interior of the model, but we were able to resolve the issue by negating our cross product. We keep these calculations as a running sum, and then return the unit vector of the sum.

Part 6: Loop subdivision for mesh upsampling

To implement loop subdivision, we followed the advice given on the spec.

  1. We first computed the new positions of each old vertex in the mesh using the given formula: \((1 - n * u)\) * original_position + u * original_neighbor_position_sum. This formula required us to consider the vertex degree, the sum of positions of all neighboring vertices and the vertex weight itself. We stored this new position at Vertex::newPosition and marked isNew as false.
  2. Then, we computed the new positions of each new vertex at the midpoint of all edges. We did this by using the formula on the spec, which required us to find positions of the vertices of the neighboring triangles: \(3/8 * (A + B) + 1/8 * (C + D)\), where A, B, C and D are positions of vertices in the neighboring triangles. We stored this new position at Edge::newPosition.
  3. After that, we used our code from Task 4 to split each original edge. For the new returned vertex, we set its position to the one stored at Edge::newPosition. Task 4 already marked this vertex as new. Then, we flip the edge if it is new and connecting a old and new vertex.
  4. Finally, we are able to update each vertex with its new position.
