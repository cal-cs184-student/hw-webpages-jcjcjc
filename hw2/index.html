Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm works by repeatedly lerping (linearly interpolating) between pairs of neighboring control points until you are left with one point. Given a parameter t, you take your n control points, lerp each neighboring pair to get n-1 new points, then repeat on those new points, and keep going until you have just one point left. That point falls on the Bezier curve at parameter t. Changing t from 0 to 1 traces out the full curve.

For the implementation, evaluateStep does one round of this algorithm. It loops through the input points, lerps each neighboring pair using (1-t)*p[i] + t*p[i+1], and returns the resulting list of intermediate points. To evaluate the full curve you would simply call this method repeatedly until one point remains.

Part 6: Loop subdivision for mesh upsampling

To implement loop subdivision, we followed the advice given on the spec.

  1. We first computed the new positions of each old vertex in the mesh using the given formula: \((1 - n * u)\) * original_position + u * original_neighbor_position_sum. This formula required us to consider the vertex degree, the sum of positions of all neighboring vertices and the vertex weight itself. We stored this new position at Vertex::newPosition and marked isNew as false.
  2. Then, we computed the new positions of each new vertex at the midpoint of all edges. We did this by using the formula on the spec, which required us to find positions of the vertices of the neighboring triangles: \(3/8 * (A + B) + 1/8 * (C + D)\), where A, B, C and D are positions of vertices in the neighboring triangles. We stored this new position at Edge::newPosition.
  3. After that, we used our code from Task 4 to split each original edge. For the new returned vertex, we set its position to the one stored at Edge::newPosition. Task 4 already marked this vertex as new. Then, we flip the edge if it is new and connecting a old and new vertex.
  4. Finally, we are able to update each vertex with its new position.
We ran into some issues when splitting edges, but we realized that we were not marking new edges as new before returning, which made our loop division output some interesting results.

Some more examples:

We noticed that the corners and edges of the shape become more curved / less sharp because in our algorithm, we end up taking the weighted average of the neighboring vertices when we are creating the new vertices. To preserve some of these sharp edges, we can presplit them, which introduces more vertices where the sharp edges are, so that neighboring vertices are closer to each other prior to loop subdivision.

When loading dae/cube.dae and performing iterations of loop subdivision, the resulting shape is noticeably asymmetric, like it is above. This is because there is already asymmetry before loop subdivision. Each face of the cube is split into two triangles by a single edge along the diagonal, and all those diagonals go in the same direction.

To make subdivision symmetric, I split the diagonal edge on each of the six cube faces. Splitting a diagonal edge inserts a new vertex at its midpoint and creates a symmetric "X" pattern of four triangles on each face, so additional subdivision then creates a more symmetric, rounded shape as seen below.
